## 1.0.109 — Fixes and local-patch recovery on updates

### Bug Fixes
- **[pi daemon]** `_start_pi_dev_daemon()` now creates `pi-agent-artifacts/` directory before running the container harness, preventing permission-denied failures when the container entrypoint writes job results.

### Feature: Local Patch Preservation on Updates
When a PAC update is applied (online or uploaded), the platform now:
1. **Before applying**: Generates a diff of all local changes since the previously installed version (compares the matching `extracted-{timestamp}/` directory to the current `app/`).
2. **Stores the diff**: Written to `updates/local-patches/from-{old}-to-{new}.patch` and also zipped as `updates/local-patch-from-{old}-to-{new}.zip` for recovery.
3. **After applying the new version**: Reapplies that diff automatically on top.
4. **Reports success/failure**: If the diff cannot be applied cleanly, the failure is returned in the API response (`local_diff` + `diff_applied` fields) so the UI can show the user which files failed and why.

This means local modifications (custom web UI changes, custom scripts, config overrides) survive across version updates. If a patch fails to apply, the user sees which files conflicted and can resolve manually.

### Files Changed
- `pi_agent_platform/updates/__init__.py` — `apply_release_zip()` now accepts `previous_version` arg, runs diff generation + diff application as part of the update flow.
- `pi_agent_platform/updates/diff_utils.py` — New module: `generate_diff()`, `apply_diff()`, `find_extracted_for_version()`, `get_diff_file()`, `diff_summary()`, `build_diff_between_versions()`.
- `pi_agent_platform/api/main.py` — `_start_pi_dev_daemon()` creates `pi-agent-artifacts/` before container launch.

### Model Balancer Skill
- Background skill that scores every configured model on speed (0-100) and config (0-100).
- Runs on startup, every 60 seconds, and after session completion.
- Speed score: proxies based on provider responsiveness and session throughput.
- Config score: evaluates context window, GPU offload, flash attention, batch size, inference settings for LM Studio models.
- Non-intrusive: only dots on cards (mouseover explanation) and a collapsible recommendations panel.
- Skill definition at `skills/model-balancer/SKILL.md`.

## Entity model (canonical definitions, 2026-05-07)

**Endpoint** — A machine that code can be executed on or from. Has a hostname/IP, a runtime (podman/docker/native), and can run agents via runner processes.

**Provider** — Provides models. Examples: LM Studio (local), OpenAI (cloud), Ollama. A provider exposes a list of available models and handles API communication with that backend.

**Workspace** — A predefined development environment. Can be used for operations tasks, documentation, or general working context. Has a path, a type (local/git/profile), and can be assigned to sessions or profiles.

**Profile** — Defines an agent type. Specifies which model it uses, its own context (mode/size), a default workspace for sessions created from it, which tools it may access, and what permission rules apply. Sessions are created from profiles.

**Tool** — Instructions on how to use a specific binary/command: what the binary path is, what input it expects, what output it produces, and what permissions/scoping apply. Tools are grouped into packages (tool packages).

**Plugins / Tool Packages** — A plugin is a collection of tools and metadata (name, version, description) distributed as a unit. The tools tab in PAC shows installed tool packages. This area may be expanded into a proper plugin marketplace in future.

### Token minting for external programs (2026-05-07)
- `POST /v1/auth/tokens` (admin) — mint a bearer token for any user, with optional ttl_hours. No password needed.
- `GET /v1/auth/tokens` (admin) — list active tokens (user + expiry, no token values)
- `DELETE /v1/auth/tokens/{token}` (admin) — revoke a token
- Use case: initial admin setup, CLI tools, CI pipelines, external programs connecting to PAC

- After setup: token stored, user chip shown, app initializes normally
