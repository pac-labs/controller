# PAC GitHub repository layout

PAC uses two GitHub repositories under `pac-labs`.

## `pac-labs/controller`

The controller repository is the source of truth for the PAC controller application. It contains the FastAPI service, web UI, release workflow, updater assets, bundled source packages, and package generation scripts.

Release assets are generated by `.github/workflows/pac-release.yml`:

- `pac-full-<version>.zip`
- `pac-patch-<version>.zip`
- `PAC_RELEASE_MANIFEST.json`
- `PAC_CHANGELOG.json`
- `pac-packages-seed-<version>.zip`

## `pac-labs/packages`

The packages repository is a tiny package catalog for the PAC Source Library. It mirrors the source roots used by the controller:

- `binaries/`
- `containers/`
- `plugins/`
- `scripts/`
- `docs/`

Each package/component can include `pac-component.json` with metadata:

```json
{
  "id": "pac-agent",
  "title": "PAC Wrapper",
  "description": "PAC-side wrapper around the pi.dev runtime.",
  "kind": "binary",
  "version": "1.0.98",
  "maintainers": ["pac-labs"],
  "repository": "https://github.com/pac-labs/packages",
  "source_path": "binaries/pac-agent",
  "tags": ["pac", "pi.dev"]
}
```

The generated repository also contains `packages.json`, which indexes all discovered component metadata.

## Seeding workflow

`.github/workflows/seed-packages.yml` checks out `pac-labs/controller`, checks out `pac-labs/packages`, runs `scripts/generate-package-repo.py`, and commits the generated mirror.

For cross-repository writes, add a repository secret named `PAC_PACKAGES_TOKEN` to `pac-labs/controller`. The token needs write access to `pac-labs/packages`.
